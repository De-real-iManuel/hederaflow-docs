## Core User Journeys

### 1. New User Onboarding

<Steps>
  <Step title="Registration">
    User creates account with email/password
    - Backend creates Hedera account automatically
    - JWT token issued for authentication
    - User redirected to meter registration
  </Step>
  
  <Step title="Meter Registration">
    User registers their electricity meter
    - Validates meter ID format (8-15 chars)
    - Selects utility provider from database
    - Chooses band classification (A-E)
    - Meter linked to user account
  </Step>
  
  <Step title="First Verification">
    User captures first meter reading
    - Photo uploaded and processed
    - OCR extracts reading value
    - Baseline established (no bill generated)
    - Verification logged to HCS
  </Step>
</Steps>

**Time to complete**: ~3 minutes

### 2. Monthly Bill Verification

<Steps>
  <Step title="Capture Reading">
    User takes photo of meter
    - Camera interface with guidelines
    - Client-side OCR for instant feedback
    - Image uploaded to backend
  </Step>
  
  <Step title="AI Processing">
    Backend validates reading
    - Google Vision API extracts text (if needed)
    - Fraud detection analyzes image
    - Consumption calculated vs. previous reading
    - Confidence score computed
  </Step>
  
  <Step title="Bill Generation">
    System calculates charges
    - Fetches tariff rates from database
    - Applies band-specific pricing
    - Adds taxes and platform fee
    - Converts to HBAR at current rate
  </Step>
  
  <Step title="Blockchain Logging">
    Verification logged to HCS
    - Regional topic selected by country
    - Immutable audit trail created
    - Sequence number returned
    - Viewable on HashScan
  </Step>
</Steps>

**Time to complete**: ~30 seconds

### 3. Bill Payment

<Steps>
  <Step title="Review Bill">
    User views bill details
    - Consumption breakdown
    - Tariff rates applied
    - HBAR amount required
    - Exchange rate displayed
  </Step>
  
  <Step title="Prepare Payment">
    System locks exchange rate
    - 5-minute rate lock created
    - 2% volatility buffer applied
    - Transaction details prepared
    - HashPack connection initiated
  </Step>
  
  <Step title="Sign Transaction">
    User approves in HashPack
    - Transaction details shown
    - User confirms in wallet
    - Transaction submitted to Hedera
    - Transaction ID returned
  </Step>
  
  <Step title="Verify & Confirm">
    Backend verifies payment
    - Mirror Node query for transaction
    - Amount validation (±1% tolerance)
    - Bill status updated to PAID
    - Payment logged to HCS
    - PDF receipt generated
  </Step>
</Steps>

**Time to complete**: ~1 minute

## Demo Highlights

### 1. Real-Time OCR Processing

```typescript
// Client-side OCR with Tesseract.js
const { data: { text, confidence } } = await Tesseract.recognize(
  imageFile,
  'eng',
  { logger: m => console.log(m) }
);

// Fallback to server-side Google Vision API
if (confidence < 0.90) {
  const result = await verificationApi.scanMeter(imageFile);
  // Google Vision typically achieves 95%+ confidence
}
```

### 4. Dynamic Tariff Calculation

```python
# Nigeria NERC-compliant tariff (Band A example)
tariff = {
    'base_rate': 225.0,  # NGN per kWh
    'vat': 0.075,        # 7.5%
    'distribution_charge': 15.0,
    'service_charge': 1500.0
}

# Calculation
consumption = 150  # kWh
base_charge = consumption * 225.0  # 33,750 NGN
vat = base_charge * 0.075          # 2,531.25 NGN
total = base_charge + vat + 1500   # 37,781.25 NGN
```

**Demo**: Register meters with different bands to see pricing differences
