## Design Philosophy

Hedera Flow is built with three core principles:

1. **Blockchain-First**: Every critical operation (verification, payment) is logged to Hedera HCS for immutability
2. **Fail-Safe**: Graceful degradation when external services fail (OCR, IPFS, exchange rates)
3. **Production-Ready**: Enterprise patterns (connection pooling, caching, rate limiting, error handling)

## Architecture Decisions

### 1. Why FastAPI over Flask/Django?

**Decision**: FastAPI with async/await

**Rationale**:
- **Performance**: 2-3x faster than Flask for I/O-bound operations (OCR, blockchain calls)
- **Type Safety**: Pydantic models catch errors at development time
- **Auto-Documentation**: OpenAPI/Swagger UI generated automatically
- **Async Support**: Native async/await for concurrent Hedera operations

```python
# Async endpoint with dependency injection
@router.post("/verify")
async def create_verification(
    image: UploadFile = File(...),
    db: Session = Depends(get_db),
    current_user: dict = Depends(get_current_user)
):
    # Concurrent operations
    ocr_task = asyncio.create_task(ocr_service.extract_reading(image_bytes))
    fraud_task = asyncio.create_task(fraud_service.analyze(image_bytes))
    
    ocr_result, fraud_result = await asyncio.gather(ocr_task, fraud_task)
```

### 2. PostgreSQL + Redis Hybrid Storage

**Decision**: PostgreSQL for persistent data, Redis for ephemeral data

**Rationale**:
- **PostgreSQL**: ACID compliance for financial data (bills, payments)
- **Redis**: Sub-millisecond access for hot data (exchange rates, rate locks)
- **Cost**: Supabase free tier (500MB), Upstash free tier (10K commands/day)

**Data Distribution**:
```python
# PostgreSQL: Long-term storage
- users, meters, verifications, bills, payments

# Redis: Short-term caching
- exchange_rates (5-min TTL)
- tariff_data (1-hour TTL)
- rate_locks (5-min exact TTL)
- api_rate_limits (1-min sliding window)
```

### 3. Regional HCS Topics

**Decision**: 5 regional topics instead of 1 global topic

**Rationale**:
- **Scalability**: Distributes load across topics (10K msg/sec per topic)
- **Compliance**: Data residency requirements (GDPR, local regulations)
- **Performance**: Reduced query time when filtering by region
- **Cost**: Same cost as single topic (HCS pricing is per-message)

**Topic Routing**:
```python
HCS_TOPICS = {

def get_topic_for_country(country_code: str) -> str:
```

### 4. Dual OCR Strategy

**Decision**: Client-side Tesseract.js + server-side Google Vision API

**Rationale**:
- **Cost**: Tesseract is free, Google Vision costs $1.50/1000 images
- **Privacy**: Client-side processing keeps images on device
- **Accuracy**: Google Vision as fallback for low-confidence results
- **Latency**: Client-side is instant, server-side adds 2-3s

**Implementation**:
```typescript
// Client-side OCR (free, instant)
const clientResult = await Tesseract.recognize(image, 'eng');

if (clientResult.confidence >= 0.90) {
  // Use client result
  submitVerification({ ocr_reading: clientResult.text, ocr_confidence: clientResult.confidence });
} else {
  // Fallback to server-side Google Vision
  submitVerification({ image: imageFile });  // Backend will process
}
```

### 5. Payment Rate Locking

**Decision**: 5-minute rate lock with 2% volatility buffer

**Rationale**:
- **User Protection**: Prevents price changes during payment flow
- **Volatility Buffer**: 2% buffer covers typical HBAR price swings
- **Expiration**: 5 minutes is enough for wallet approval, not too long to hold stale rates

**Implementation**:
```python
# Create rate lock in Redis
rate_lock = {
    'hbar_price': 0.05,

    'buffer_percentage': 2.0,
    'locked_at': datetime.utcnow(),

}

redis_client.setex(
```

## Security Considerations

### 1. JWT Token Security

**Implementation**:
```python
# Token generation with 30-day expiration
def create_access_token(user_id: str) -> str:
    payload = {
        'sub': user_id,
        'exp': datetime.utcnow() + timedelta(days=30),
        'iat': datetime.utcnow(),
        'type': 'access'
    }
    return jwt.encode(payload, settings.jwt_secret_key, algorithm='HS256')

# Token validation middleware
async def get_current_user(token: str = Depends(oauth2_scheme)) -> dict:
    try:
        payload = jwt.decode(token, settings.jwt_secret_key, algorithms=['HS256'])
        user_id = payload.get('sub')
        if not user_id:
            raise HTTPException(401, "Invalid token")
        return {'user_id': user_id}
    except JWTError:
        raise HTTPException(401, "Invalid token")
```

**Security Measures**:
- HS256 algorithm (symmetric, fast)
- 30-day expiration (balance between UX and security)
- Token stored in localStorage (XSS risk mitigated by CSP)
- Future: Refresh tokens + token blacklisting in Redis

## Hedera Integration Deep-Dive

### 1. HCS Message Submission

**Optimized Implementation**:
```python
async def submit_hcs_message(topic_id: str, message: dict) -> dict:
    # Serialize message
    message_json = json.dumps(message)
    message_bytes = message_json.encode('utf-8')
    
    # Create transaction
    transaction = (
        TopicMessageSubmitTransaction()
        .setTopicId(TopicId.fromString(topic_id))
        .setMessage(message_bytes)
        .setMaxTransactionFee(Hbar(1))  # Cap fee at 1 HBAR
    )
    
    # Execute with retry logic
    max_retries = 3
    for attempt in range(max_retries):
        try:
            response = transaction.execute(client)
            receipt = response.getReceipt(client)
            
            return {
                'topic_id': topic_id,
                'sequence_number': receipt.topicSequenceNumber,
                'consensus_timestamp': receipt.consensusTimestamp
            }
        except Exception as e:
            if attempt == max_retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)  # Exponential backoff
```

**Cost**: ~$0.0001 per message (negligible)

### 2. Mirror Node Transaction Verification

**Comprehensive Validation**:
```python
async def verify_transaction(tx_id: str, expected_amount: Decimal) -> dict:
    # Query Mirror Node API
    url = f"https://testnet.mirrornode.hedera.com/api/v1/transactions/{tx_id}"
    response = await httpx.get(url)
    
    if response.status_code == 404:
        raise TransactionNotFoundError(f"Transaction {tx_id} not found")
    
    data = response.json()
    tx = data['transactions'][0]
    
    # Validate status
    if tx['result'] != 'SUCCESS':
        raise TransactionFailedError(f"Transaction failed: {tx['result']}")
    
    # Validate amount
    actual_amount = Decimal(tx['transfers'][0]['amount']) / 100_000_000  # Tinybars to HBAR
    tolerance = expected_amount * Decimal('0.01')  # 1% tolerance
    
    if abs(actual_amount - expected_amount) > tolerance:
        raise AmountMismatchError(
            f"Expected {expected_amount} HBAR, got {actual_amount} HBAR"
        )
    
    return {
        'amount_hbar': actual_amount,
        'consensus_timestamp': tx['consensus_timestamp']
    }
```

## Error Handling Patterns

### 1. Graceful Degradation

**OCR Fallback Chain**:
```python
def extract_reading(image_bytes: bytes) -> dict:
    try:
        # Try Google Vision API
        return google_vision_ocr(image_bytes)
    except VisionAPIQuotaExceeded:
        logger.warning("Vision API quota exceeded, using Tesseract")
        return tesseract_ocr(image_bytes)
    except VisionAPIUnavailable:
        logger.error("Vision API unavailable, using Tesseract")
        return tesseract_ocr(image_bytes)
    except Exception as e:
        logger.error(f"All OCR methods failed: {e}")
        return {'reading': 0.0, 'confidence': 0.0, 'error': str(e)}
```

### 2. Transaction Rollback

**Database Transactions**:
```python
@router.post("/verify")
async def create_verification(...):

        # Start transaction
        db.begin()
        
        # Create verification
```

### 3. Circuit Breaker Pattern

**External API Protection** (Future):
```python
class CircuitBreaker:
    def __init__(self, failure_threshold: int = 5, timeout: int = 60):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.timeout = timeout
        self.last_failure_time = None
        self.state = 'CLOSED'  # CLOSED, OPEN, HALF_OPEN
    
    async def call(self, func, *args, **kwargs):
        if self.state == 'OPEN':
            if time.time() - self.last_failure_time > self.timeout:
                self.state = 'HALF_OPEN'
            else:
                raise CircuitBreakerOpenError("Circuit breaker is open")
        
        try:
            result = await func(*args, **kwargs)
            self.on_success()
            return result
        except Exception as e:
            self.on_failure()
            raise
```
